# 数据库迁移步骤说明

本文档说明如何清理旧格式新闻数据，并将新格式的新闻存入数据库。

## 📋 前置准备

1. **确保 MongoDB 正在运行**
   - 本地 MongoDB：确保服务已启动
   - 云数据库：确保连接字符串可用

2. **配置环境变量**
   - 在 `backend` 目录下创建 `.env` 文件（如果还没有）
   - 添加以下内容：
   ```
   MONGODB_URI=mongodb://localhost:27017/
   MONGODB_DB_NAME=ai_news
   PORT=8000
   ```
   - 如果是远程数据库，格式为：`mongodb://用户名:密码@主机:端口/`

3. **安装 Python 依赖**
   - 确保已安装 `pymongo`：
   ```bash
   pip install pymongo
   ```

---

## 步骤2：清理旧数据

### 方法一：使用清理脚本（推荐）

1. **运行清理脚本**
   ```bash
   cd backend
   node scripts/clearOldNews.js
   ```

2. **脚本会：**
   - 自动连接 MongoDB
   - 查找所有旧格式的新闻（没有 `title_en` 或 `title_zh` 字段的）
   - 显示找到的数量
   - 删除所有旧格式数据

3. **注意事项：**
   - 删除操作不可逆，建议先备份数据库
   - 如果只想查看数量而不删除，可以临时注释掉脚本中的 `deleteMany` 操作

### 方法二：使用 MongoDB Shell

```bash
# 连接到 MongoDB
mongosh "你的MongoDB连接字符串"

# 切换到你的数据库
use ai_news

# 查看旧数据数量
db.news.countDocuments({ $or: [{ title_en: { $exists: false } }, { title_zh: { $exists: false } }] })

# 删除旧数据
db.news.deleteMany({ $or: [{ title_en: { $exists: false } }, { title_zh: { $exists: false } }] })
```

## 步骤3：修改爬虫代码，写入数据库

### 已完成的修改

我已经修改了 `backend/CrawTasks/crawer_bbc.py`，主要改动：

1. **添加了 MongoDB 连接功能**
   - `get_mongodb_client()`: 连接 MongoDB
   - `save_to_mongodb()`: 保存新闻到数据库

2. **修改了 `crawl_and_save()` 函数**
   - 新增参数 `save_to_db=True`（默认保存到数据库）
   - 如果 `save_to_db=False`，则保存到 jsonl 文件（保留原有功能）

3. **自动去重**
   - 根据 `link` 字段判断是否已存在
   - 如果已存在，跳过不重复插入

### 使用方法

1. **设置环境变量**（在 Python 脚本运行前）
   ```bash
   # Windows PowerShell
   $env:MONGODB_URI="mongodb://localhost:27017/"
   $env:MONGODB_DB_NAME="ai_news"
   
   # Windows CMD
   set MONGODB_URI=mongodb://localhost:27017/
   set MONGODB_DB_NAME=ai_news
   
   # Linux/Mac
   export MONGODB_URI="mongodb://localhost:27017/"
   export MONGODB_DB_NAME="ai_news"
   ```

2. **运行爬虫**
   ```bash
   cd backend/CrawTasks
   python crawer_bbc.py
   ```

3. **参数说明**
   ```python
   # 保存到数据库（默认）
   crawl_and_save(llm_model='gpt', save_to_db=True)
   
   # 保存到文件（备用）
   crawl_and_save(llm_model='gpt', save_to_db=False)
   ```

### 验证数据

运行爬虫后，可以通过以下方式验证：

1. **使用 MongoDB Shell**
   ```bash
   mongosh "你的MongoDB连接字符串"
   use ai_news
   db.news.find().limit(1).pretty()
   ```

2. **使用后端 API**
   ```bash
   # 启动后端服务器

   curl http://localhost:8000/api/news?page=1&limit=5
   ```

## ⚠️ 注意事项

1. **数据备份**：清理旧数据前，建议先备份数据库
2. **环境变量**：确保 Python 脚本能读取到 MongoDB 连接字符串
3. **去重机制**：爬虫会根据 `link` 自动去重，不会重复插入相同链接的新闻
4. **时间格式**：`postedAt` 和 `crawledAt` 会自动转换为 MongoDB 的 Date 类型

## 🐛 常见问题

**Q: 爬虫报错 "MongoDB 连接失败"**
- 检查 MongoDB 服务是否运行
- 检查环境变量 `MONGODB_URI` 是否正确
- 检查网络连接和防火墙设置

**Q: 数据没有保存到数据库**
- 检查是否有重复的 `link`（会被自动跳过）
- 查看控制台输出的错误信息
- 确认数据库名称是否正确

**Q: API 返回的数据格式不对**
- 确认已更新 `News.js` 模型（步骤1已完成）
- 重启后端服务器
- 检查数据库中的实际数据格式
