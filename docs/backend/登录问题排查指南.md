## 排查步骤

### 步骤1：确认测试账号已创建

运行测试账号创建脚本：
```bash
cd backend
node scripts/createTestUser.js
```

**默认测试账号：**
- 邮箱：`test@example.com`
- 密码：`Test@1234`

**如果显示"用户已存在"**，说明账号已创建，可以直接使用。

---

### 步骤2：检查后端服务是否运行

```bash
cd backend
node server.js
```

应该看到：
```
✅ Connected to MongoDB
🚀 Backend server is running on port 8000
```

**如果 MongoDB 连接失败**：
- 检查 MongoDB 是否运行：`netstat -an | findstr 27017`
- 检查 `.env` 文件中的 `MONGODB_URI` 是否正确

---

### 步骤3：检查前端 API 地址

打开 `frontend/services/api.ts`，确认：
```typescript
const API_BASE_URL = (import.meta as any).env?.VITE_API_BASE_URL || 'http://localhost:8000/api';
```

**确保后端运行在 `http://localhost:8000`**

---

### 步骤4：测试登录接口（使用 curl 或 Postman）

**使用 curl：**
```bash
curl -X POST http://localhost:8000/api/auth/login \
  -H "Content-Type: application/json" \
  -d "{\"email\":\"test@example.com\",\"password\":\"Test@1234\"}"
```

**应该返回：**
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": "...",
    "email": "test@example.com",
    "name": "Test User"
  }
}
```

**如果返回错误：**
- `400`：邮箱或密码为空
- `401`：邮箱或密码错误
- `500`：服务器内部错误（查看后端控制台日志）

---

### 步骤5：检查浏览器控制台

1. 打开浏览器开发者工具（F12）
2. 切换到 "Network"（网络）标签
3. 尝试登录
4. 查看 `/api/auth/login` 请求：
   - **状态码**：应该是 200
   - **请求体**：应该是 JSON 格式
   - **响应**：应该包含 `token` 和 `user`

**常见错误：**
- `CORS error`：检查后端是否启用了 CORS（已启用）
- `Failed to fetch`：后端服务未运行或地址错误
- `401 Unauthorized`：邮箱或密码错误

---

### 步骤6：检查后端控制台日志

登录时，后端控制台应该显示：
- 如果成功：无错误信息
- 如果失败：会显示错误信息

**查看错误信息：**
```javascript
// backend/server.js 第119-121行
catch (err) {
  console.error('Error login:', err);  // 这里会打印错误
  res.status(500).json({ error: '登录失败' });
}
```

## 常见问题及解决方案

### 问题1：登录失败，显示"邮箱或密码错误"

**可能原因：**
1. 账号未创建
2. 密码输入错误
3. 邮箱输入错误

**解决方案：**
1. 运行 `node scripts/createTestUser.js` 创建账号
2. 确认使用正确的邮箱和密码
3. 检查邮箱是否有多余空格

---

### 问题2：登录失败，显示"登录失败"（500错误）

**可能原因：**
1. MongoDB 未连接
2. JWT_SECRET 未设置
3. bcrypt 模块未安装

**解决方案：**
1. 检查 MongoDB 连接（步骤2）
2. 在 `.env` 文件中添加 `JWT_SECRET=your-secret-key`
3. 运行 `npm install` 安装依赖

---

### 问题3：前端显示"登录失败"，但后端无错误

**可能原因：**
1. 前端 API 地址错误
2. CORS 问题
3. 网络连接问题

**解决方案：**
1. 检查前端 API 地址配置
2. 检查后端 CORS 设置（已启用）
3. 检查防火墙设置

---

### 问题4：登录成功但 Token 未保存

**可能原因：**
1. localStorage 被禁用
2. 浏览器隐私模式

**解决方案：**
1. 检查浏览器是否允许 localStorage
2. 退出隐私模式

---
