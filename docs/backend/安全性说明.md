# 安全性说明

## 密码存储安全性 ✅

### 当前实现
- **密码使用 bcrypt 哈希存储**，不会存储明文密码
- bcrypt 是业界标准的密码哈希算法，具有以下特点：
  - 单向加密：无法从哈希值反推原始密码
  - 加盐（Salt）：每个密码都有唯一的盐值
  - 可调节成本：当前设置为 10 轮（安全且性能平衡）

### 代码位置
```javascript
// backend/server.js 第70行
const passwordHash = await bcrypt.hash(password, 10);

// 登录验证时（第105行）
const ok = await bcrypt.compare(password, user.passwordHash);
```

### 安全性评估
✅ **密码存储：非常安全**
- 即使数据库泄露，攻击者也无法直接获取用户密码
- 需要暴力破解，bcrypt 的 10 轮成本使得破解非常困难

## Token 存储安全性 ⚠️

### 当前实现
- JWT Token 存储在浏览器的 `localStorage` 中
- Token 有效期：7 天

### 潜在风险
1. **XSS 攻击**：如果网站存在 XSS 漏洞，恶意脚本可以读取 localStorage 中的 token
2. **Token 泄露**：如果用户电脑被恶意软件感染，可能读取 localStorage

### 改进建议（生产环境）

#### 方案1：使用 httpOnly Cookie（推荐）
```javascript
// 后端设置 cookie
res.cookie('token', token, {
  httpOnly: true,  // 防止 JavaScript 访问
  secure: true,    // 仅 HTTPS 传输
  sameSite: 'strict',  // 防止 CSRF
  maxAge: 7 * 24 * 60 * 60 * 1000  // 7天
});
```

#### 方案2：使用 sessionStorage（临时改进）
- sessionStorage 在标签页关闭时自动清除
- 比 localStorage 稍安全，但仍可能被 XSS 攻击

#### 方案3：短期 Token + Refresh Token
- Access Token：短期（15分钟）
- Refresh Token：长期（7天），存储在 httpOnly cookie

## HTTPS 传输安全性 ⚠️

### 当前状态
- 开发环境通常使用 HTTP（`http://localhost:8000`）
- **生产环境必须使用 HTTPS**

### 为什么需要 HTTPS
1. **防止中间人攻击**：加密传输，防止密码和 token 被窃听
2. **防止数据篡改**：确保数据完整性
3. **浏览器安全策略**：现代浏览器对 HTTP 网站有安全限制

### 部署建议
- 使用 Nginx/Apache 反向代理 + Let's Encrypt 免费 SSL 证书
- 或使用云服务商提供的 HTTPS（如 Vercel、Netlify）

## 数据库安全性 ✅

### 当前实现
- MongoDB 连接字符串存储在 `.env` 文件中
- `.env` 文件已添加到 `.gitignore`（不会被提交到 Git）

### 安全建议
1. ✅ **不要将 `.env` 提交到 Git**
2. ✅ **生产环境使用环境变量**，不要硬编码
3. ⚠️ **MongoDB 访问控制**：
   - 生产环境启用 MongoDB 认证
   - 使用强密码
   - 限制 IP 访问（如果可能）

---

## API 接口安全性 ✅

### 当前实现
- 需要认证的接口使用 `authRequired` 中间件
- JWT Token 验证

### 安全建议
1. ✅ **Token 过期时间**：当前 7 天，可根据需求调整
2. ⚠️ **速率限制**：建议添加 API 请求频率限制，防止暴力破解
3. ⚠️ **登录失败次数限制**：防止暴力破解密码
