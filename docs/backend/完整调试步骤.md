# 🔍 登录问题完整调试步骤

## 第一步：确认测试账号存在

运行测试脚本：
```bash
cd backend
node scripts/testLogin.js
```

**期望输出：**
```
✅ MongoDB 连接成功
✅ 找到用户
✅ 密码验证成功！
```

**如果显示"用户不存在"**：
```bash
node scripts/createTestUser.js
```

---

## 第二步：检查后端服务

1. **启动后端服务**：
```bash
cd backend
node server.js
```

2. **确认看到以下输出**：
```
✅ Connected to MongoDB
🚀 Backend server is running on port 8000
```

3. **如果没有看到 ✅ Connected to MongoDB**：
   - 检查 MongoDB 是否运行：`netstat -an | findstr 27017`
   - 检查 `.env` 文件中的 `MONGODB_URI`

---

## 第三步：测试后端登录接口（使用 curl）

打开**新的命令行窗口**（保持后端服务运行），运行：

```bash
curl -X POST http://localhost:8000/api/auth/login -H "Content-Type: application/json" -d "{\"email\":\"test@example.com\",\"password\":\"Test@1234\"}"
```

**期望返回：**
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": "...",
    "email": "test@example.com",
    "name": "Test User"
  }
}
```

**如果返回错误**：
- 查看后端控制台的详细日志（我添加了详细的日志输出）
- 根据错误信息排查

---

## 第四步：检查前端

1. **打开浏览器开发者工具**（F12）
2. **切换到 Network（网络）标签**
3. **尝试登录**
4. **查看 `/api/auth/login` 请求**：

   **请求详情：**
   - Method: `POST`
   - URL: `http://localhost:8000/api/auth/login`
   - Headers: 应该包含 `Content-Type: application/json`
   - Request Payload: 应该包含 `{"email":"test@example.com","password":"Test@1234"}`

   **响应详情：**
   - Status: 应该是 `200 OK`
   - Response: 应该包含 `token` 和 `user`

---

## 第五步：查看后端控制台日志

登录时，后端控制台应该显示：
```
🔐 收到登录请求
请求体: {"email":"test@example.com","password":"Test@1234"}
🔍 查找用户: test@example.com
✅ 找到用户，验证密码...
✅ 密码验证通过，生成 token...
✅ 登录成功
```

**如果看到错误**：
- `❌ 邮箱或密码为空` → 检查前端请求格式
- `❌ 用户不存在` → 运行 `node scripts/createTestUser.js`
- `❌ 密码错误` → 检查密码是否正确
- `❌ 登录错误` → 查看错误堆栈

---

## 常见问题排查

### 问题1：后端控制台没有任何输出

**原因**：请求没有到达后端

**解决**：
1. 检查后端服务是否运行
2. 检查前端 API 地址是否正确（`http://localhost:8000/api`）
3. 检查 CORS 设置（已启用，应该没问题）

---

### 问题2：显示"用户不存在"

**解决**：
```bash
cd backend
node scripts/createTestUser.js
```

然后再次测试：
```bash
node scripts/testLogin.js
```

---

### 问题3：显示"密码错误"

**可能原因**：
1. 密码输入错误
2. 用户创建时密码哈希有问题

**解决**：
1. 确认密码是 `Test@1234`（注意大小写和特殊字符）
2. 删除旧用户，重新创建：
```bash
# 使用 MongoDB Shell
mongosh "mongodb://localhost:27017/"
use ai_news
db.users.deleteOne({ email: "test@example.com" })
exit

# 重新创建
node scripts/createTestUser.js
```

---

### 问题4：前端显示"登录失败"但后端无错误

**可能原因**：
1. 网络连接问题
2. CORS 问题
3. 前端错误处理问题

**解决**：
1. 检查浏览器控制台的 Network 标签，查看实际返回
2. 检查后端控制台是否有日志输出
3. 尝试使用 curl 直接测试后端接口

---

### 问题5：CORS 错误

**解决**：
后端已经启用了 CORS（`app.use(cors())`），如果还有问题：
1. 检查后端服务是否运行
2. 检查前端请求的 URL 是否正确
