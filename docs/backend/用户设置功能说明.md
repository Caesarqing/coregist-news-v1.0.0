# 用户推送设置功能说明

## 功能概述

用户可以在首页设置新闻推送偏好，包括：
- **推送日期**：选择一周中的哪些天接收推送
- **推送时间**：设置每天推送的具体时间
- **推送数量**：每次推送的新闻条数（1-20条）
- **新闻关键词**：设置感兴趣的新闻主题关键词

这些设置会自动保存到数据库，下次登录时会自动加载。

---

## 数据库结构

### User 模型扩展

在 `backend/models/User.js` 中添加了 `pushSettings` 字段：

```javascript
pushSettings: {
  pushDays: ['monday', 'wednesday', 'friday'],  // 推送日期数组
  pushTimes: ['08:00', '18:00'],                // 推送时间数组
  pushCount: 5,                                 // 推送数量（1-20）
  everyday: false,                               // 是否每天推送
  keywords: ['科技', 'AI'],                      // 关键词数组
}
```

## API 接口

### 1. 获取用户设置

**接口：** `GET /api/user/settings`

**认证：** 需要登录（Bearer Token）

**响应：**
```json
{
  "pushSettings": {
    "pushDays": ["monday", "wednesday", "friday"],
    "pushTimes": ["08:00", "18:00"],
    "pushCount": 5,
    "everyday": false,
    "keywords": ["科技", "AI"]
  }
}
```

---

### 2. 更新用户设置

**接口：** `PUT /api/user/settings`

**认证：** 需要登录（Bearer Token）

**请求体：**
```json
{
  "pushSettings": {
    "pushDays": ["monday", "wednesday", "friday"],
    "pushTimes": ["08:00", "18:00"],
    "pushCount": 5,
    "everyday": false,
    "keywords": ["科技", "AI", "新闻"]
  }
}
```

**响应：**
```json
{
  "ok": true,
  "pushSettings": {
    "pushDays": ["monday", "wednesday", "friday"],
    "pushTimes": ["08:00", "18:00"],
    "pushCount": 5,
    "everyday": false,
    "keywords": ["科技", "AI", "新闻"]
  }
}
```

## 前端实现

### 自动加载设置

当用户打开首页时，如果已登录，会自动从服务器加载之前保存的设置：

```typescript
useEffect(() => {
  const loadSettings = async () => {
    try {
      const result = await userSettingsApi.getSettings();
      if (result.pushSettings) {
        setPushSettings(result.pushSettings);
      }
    } catch (error) {
      // 未登录时使用默认设置
    }
  };
  loadSettings();
}, []);
```

### 自动保存设置

当用户点击"确认"按钮时，设置会自动保存到数据库：

```typescript
const handleConfirm = async () => {
  const completeSettings = {
    ...pushSettings,
    keywords: keywordInput.trim().split(/\s+/)
  };
  
  // 保存到数据库
  await userSettingsApi.updateSettings(completeSettings);
  
  // 继续执行搜索逻辑
  onSearchComplete(keywords, completeSettings);
};
```

## 使用流程

1. **用户登录** → 系统自动加载之前保存的设置
2. **用户修改设置** → 在首页调整推送日期、时间、数量、关键词
3. **用户点击确认** → 设置自动保存到数据库
4. **下次登录** → 自动加载上次保存的设置

## 数据验证

### 后端验证

- `pushDays`: 必须是字符串数组
- `pushTimes`: 必须是字符串数组（格式：HH:mm）
- `pushCount`: 必须是数字，自动限制在 1-20 之间
- `everyday`: 必须是布尔值
- `keywords`: 必须是字符串数组

### 前端验证

- 推送日期：至少选择一个日期或选择"每天"
- 推送时间：至少设置一个时间
- 推送数量：1-20 之间
- 关键词：至少输入一个关键词才能确认

```javascript
{
  pushDays: ['monday', 'wednesday', 'friday'],
  pushTimes: ['08:00', '18:00'],

  keywords: []

```

---

## 注意事项

1. **需要登录**：保存设置需要用户登录，未登录时只能使用默认设置
2. **自动保存**：点击"确认"时自动保存，无需额外操作
3. **数据持久化**：设置保存在 MongoDB 数据库中，永久保存
4. **多设备同步**：同一账号在不同设备登录，设置会同步
